## Question format

Every entry follows this structure:

```markdown
### Q: Question text

**Topic:** networking | **Difficulty:** beginner | **Source:** Day 6

<details><summary>Answer</summary>

Concise answer (2–5 lines).

**Follow-up point:** Extra context an interviewer might probe.

</details>
```

## Difficulty scale

- 🟢 **Beginner** — direct recall, AZ-900 level
- 🟡 **Intermediate** — explain how/why, AZ-104 level
- 🔴 **Advanced** — scenario, trade-offs, multi-service design (AZ-305 / senior role)

## How to use it

**For focused topic revision:** open the topic file (e.g., `by-topic/networking.md`).

**For full mock interview:** open `question-bank.md` and answer 10 random questions in a row out loud.

**For the night before an interview:** review the "real-world scenario" bank.
